# 代码优化（code-optimizer）

## 目标
在不改变对外行为的前提下，对本仓库代码做有目标的优化：可读性、性能或可维护性。避免为了“优雅”进行大改或引入新复杂度。

## 适用场景
- 明确要“优化/重构/提速/降低复杂度”
- 评审或线上反馈指出坏味道（长函数、重复逻辑、热点路径）
- 显式调用：`$code-optimizer`

## 不做
- 没有收益证明的架构重写/大规模抽象
- 引入新框架/新中间件来解决局部问题
- 把 Bug 定位当成优化做（Bug 优先交给 bug-hunter）

## 输入
- 目标范围：文件/类/方法/接口/调用链（至少一个）
- 痛点描述：慢在哪里/难读在哪里/重复在哪里（最好带日志或现象）
- 约束：必须保持兼容的行为、性能指标、禁止改动点

## 输出
- 问题清单（按 P0/P1/P2，给出位置与理由）
- 最小改动方案（每项含收益、风险、回滚点）
- 验证清单（要跑的用例/测试、需要关注的回归点）

## 怎么调用
- `$code-optimizer` + 目标范围 + 现象 + 约束
