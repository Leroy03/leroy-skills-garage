# 永久记忆库（Permanent Memory）

本 skill 的目标是把“容易遗忘但反复会用到”的信息沉淀到仓库内的固定位置，形成可追溯、可检索、可复用的长期记忆。

## 何时调用

- 需求/方案定稿：输出契约、验收口径、边界条件确定
- 接口/配置确定：URL、Header、promptId、参数字段、默认值
- 工程约定明确：命名规范、目录结构、模块边界、依赖关系
- Bug 复盘：根因、修复点、验证方式、回滚策略
- 任何“下次还会问/还会踩”的信息出现时

## 沉淀位置（必须写入）

- 主记忆库：`SKILL/永久记忆库.md`
- 索引入口：`SKILL/永久记忆库_INDEX.md`

## 写入规则

- 只写“稳定事实”和“可执行结论”，不写聊天过程
- 每条记录必须包含：日期、主题、结论、证据/链接、验证方式
- 命名/配置必须可复制粘贴（不要只描述）
- 涉及密钥/Token 的内容只写“配置项名称与获取方式”，不写真实值

## 标准写入流程（执行步骤）

1) 从对话/代码变更中提取 3~10 条“必须沉淀的点”（不超过 10 条）
2) 按类别写入 `SKILL/永久记忆库.md`
3) 更新 `SKILL/永久记忆库_INDEX.md` 的索引（新增条目/更新时间）
4) 给出链接，方便直接跳转到新增内容

## 记忆分类模板（建议）

- 项目约定（命名/目录/模块边界）
- 核心能力与工具（例如：板块匹配工具、quote-sync）
- AIGC / Prompt（promptId、入参字段、输出格式）
- 向量检索/Embedding/Milvus（URL、鉴权方式、失败降级）
- 常用命令（构建/测试/本地启动）
- 已知问题与回滚策略

## 示例（写入一条）

```markdown
### 2026-03-05｜板块匹配工具：AIGC 拆解板块字段（P055912）
- 结论：quote-sync 的 SectorMatchService 在 resolve() 前先调用 AIGC(P055912) 抽取 sector 字段，再做 token 化与匹配。
- 配置：quote-sync.aigc.url / quote-sync.aigc.prompt-id / quote-sync.aigc.wind-sessionid
- 证据：链接到实现文件与测试
- 验证：mvn -f .\\SKILL\\quote-sync\\pom.xml test
```
